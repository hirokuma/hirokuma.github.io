Bitcoinはトランザクションを展開する際に手数料がいる。

銀行の手数料と同じような意味合いなのか、銀行に勤めたことがないのでよくわからない。  
Bitcoinのトランザクション手数料はブロックを発見したマイナー(miner)の総取りになる。

銀行の場合は窓口からなのかATMなのかネットからなのかで変わったり、送金額によっても変わったりするだろう。  
Bitcoinはそもそも固定額ではない。

まず、マイナーがそのトランザクションを次のブロックに入れようとするのに含めてほしいかどうか。
手数料を高くするとそのトランザクションは目立ってブロックに入る確率が高くなる。
単に高ければよいわけではない。他にもトランザクションがあるので、それらよりも「割合」が高くないと目立たない。
トランザクションにはサイズがあり、すかすかなトランザクションもあればデータが詰まったトランザクションもある。

トランザクションのサイズあたりの手数料を手数料率、fee rateと呼んでいる。
トランザクションはデータの塊だが、データの部位によってサイズの比重が異なる。
それを加味したサイズをvirtual size(vsize)とかvirtual byte(vbyte)とか呼ぶ。  
fee rateはfeeをvirtual byteで割ったsats/vbyteで表す。

比重が異なると書いたが、具体的には比重が1か0.25かである。
なのでfee rateは小数値になることがほとんどだ。
整数値で扱いたいときはvbyteではなく比重を4倍したweightという単位で扱ったりもする。
1vb = 4wuになるはずだ。

* [Transaction size calculator - Bitcoin Optech](https://bitcoinops.org/en/tools/calc-size/)

## rust-bitcoinでは

`FeeRate`がある。  
型は`u64`で単位は sat/kwu となっている。kwu = 1,000 weight unit = 250 vb。

* [FeeRate in bitcoin - Rust](https://docs.rs/bitcoin/0.32.101/bitcoin/struct.FeeRate.html)

`from_sat_per_vb_u32()`の使い勝手が良さそうなのだが引数が整数しかない。
`FeeRate`を使わずに計算したほうがやりやすそうだ。
四捨五入なのか切り捨てなのか切り上げなのか、そこら辺はあまり判断できていない。

```rust
    let fee = (fee_rate_vb * vbyte as f64 + 0.5) as u64;
```

1.0を下回るとダメだったのだが、最近はmempool.spaceでも普通に1.0未満になっているためよくわからん。
bitcoindが中継するかどうかの判断さえ満たせば良いだけといえばそうなんだが。

`println!`や`panic!`はマクロだった。`#[error]`のようなのもマクロだった  

Rust日本語訳では19.5章とかなり後ろの方だ。

* [マクロ - The Rust Programming Language 日本語版](https://doc.rust-jp.rs/book-ja/ch19-06-macros.html)

まず思ったのは、これは自分で積極的に書きたいものではないな、ということだ。

C/C++のマクロはプリプロセッサで処理するので強力というかなんというか、無法なやつらという印象がある。
マクロに騙されたことがない、という人は少なかろう。

あれに比べるとコンパイラのチェックが働くだろうが、ともかく難しそうだ。  
自分で書くよりもAIに作ってもらったほうがよさそうだし、ちゃんと作ってくれそうな気がする。
そのうち、そういうことすら考えずに済むようになるのかもしれんが、そこは気にすまい。
そうなったとしても「書けないけど読める」という状態にはしておきたい。

## 宣言型

値を返すこともできるし、返さなくても良い。  
C言語のマクロっぽい

```rust
#[macro_export]
macro_rules! maomao {
    ( $( $x:expr ),* ) => { $(println!($x);)* }
}

fn main() {
    maomao!["a", "hello"];
    maomao!("world", "b");
    maomao!{"!!", "c", "restaurant"};
}
```

実行

```log
a
hello
world
b
!!
c
restaurant
```

括弧の形も特に気にしていなさそうだ。  
`vec![...]`をカギ括弧にしているのは通例というだけなのか。

外に展開しないのであれば`#[macro_export]`もいらない。  
`$expr`以外にも[いろいろ](https://doc.rust-lang.org/reference/macros-by-example.html#metavariables)指定できる。

```rust
macro_rules! two_vec {
    ($x:literal, $y:literal) => {
        vec![$x, $y]
    }
}

fn main() {
    let v = two_vec!("a", "b");
    println!("{:?}", v);
}
```

変数宣言もできるが、マクロ内のローカル変数は外部から見えないそうだ。
見えないのは名前だけなので外から与えるとよい。なんだかだまされた気分だ。  
`const`, `fn`, `static`はルールが別なので見えるそうだ。Hygieneというらしい。

```rust
macro_rules! const_maker {
    ($t:ty, $v:tt) => { const CONST: $t = $v; };
}

macro_rules! value_maker {
    ($name:ident, $t:ty, $v:tt) => { let $name: $t = $v; };
}

fn main() {
    const_maker!(&'static str, "abc");
    println!("{}", CONST);
    value_maker!(myval, &'static str, "abc");
    println!("{}", myval);
}
```

### thiserrorでログを出したい

マクロを調べようと思ったのは`thiserror`のときにエラーログを簡単に埋め込みたかったからだ。  
`#[from]`だとユニークな情報を埋め込めないから`.map_err()`を使うようにして、
`.map_err()`の中で`error!`などでエラーログを出すがよかろうと思った。
が、せっかく`#[error]`でメッセージを決めたのだからログで同じようなことを書くのはめんどくさい。

そこでこういうマクロを書いてみた。  
`.map_err`の中では変換したエラー型を返すのだから、それをマクロで囲んでログを機械的に吐いてもらえばよかろう。

{% raw %}

```rust
macro_rules! mylog {
    ($err_variant:expr) => {{
        let err = $err_variant;
        error!("[{}:{}] {}", file!(), line!(), err);
        err
    }};
}
```

{% endraw %}

案外、悪くない。
が、ちょっとログがうざったい。  
これは一番上のところだけこのマクロを適用したのだが、`reason`という場所を特定するためのユニーク情報をむりやり埋め込んでいるので全体として長い。

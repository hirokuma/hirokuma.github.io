Bitcoin関係の関数で引数にアドレス文字列(`&str`)を取るようにしていた。  
それとは別の関数でrust-bitcoinの`Address`型を返す関数を作っていた。  
それなら、その関数は`&str`も`&Address`も受け取るようにすると見栄えが良いんじゃなかろうかと思った。

文字列を`Address`型にするのはだいたいこんな書き方になる。

```rust
let addr: Address<NetworkUnchecked> = addr.parse()?;
let addr: Address = addr.require_network(network)?;
```

これをうまいこと使えばよいとは思うのだが、自分では思いつかないのでDeepWikiに聞く。

* [DeepWiki](https://deepwiki.com/search/addressstrstraddress_bc21b720-2b73-401b-96f1-cbe456aac25c?mode=fast)

```rust
struct AddressWrapper(Address<NetworkUnchecked>);  
  
impl From<Address> for AddressWrapper {  
    fn from(addr: Address) -> Self {  
        AddressWrapper(addr.as_unchecked().clone())  
    }  
}  
  
impl From<&str> for AddressWrapper {  
    fn from(s: &str) -> Self {  
        AddressWrapper(s.parse().unwrap())  
    }  
}
```

このコードはそのままでは動かない。。。

せっかくなので勉強しよう。

## FromStr

`std::str::FromStr`はトレイトである。

```rust
pub const trait FromStr: Sized {
    fn from_str(s: &str) -> Result<Self, Self::Err>;
}
```

`bool`については同じファイルに定義があった。
これはわかりやすい。

```rust
impl FromStr for bool {
    type Err = ParseBoolError;
    fn from_str(s: &str) -> Result<bool, ParseBoolError> {
        match s {
            "true" => Ok(true),
            "false" => Ok(false),
            _ => Err(ParseBoolError),
        }
    }
}
```

## From

DeepWikiのコードに`FromStr`が含まれていたので調べたが、`Address`は`FromStr`を持っているので`.parse()`すればよいだけである。
つまりこのコードの中では`FromStr`は使わなくて良い。

`struct AddressWrapper(Address<NetworkUnchecked>)`というnewtypeパターンのタプル型を用意している。  
それと`From`トレイトを使って、`&str`から`AddressWrapper`に変換する場合と、`Address`から`AddressWrapper`に変換する場合を実装しようとしているのだろう。

せっかくの`From`トレイトなのだが、引数が1つしか取ることができない。

```rust
pub const trait From<T>: Sized {
    fn from(value: T) -> Self;
}
```

`Address`型にするためには`Network`がないとできないので`&str`から直接`Address`にすることができない。  
それに引きずられて`Address`型を渡したとしても同じ型にするため`as_unchecked()`で`Network`情報を取り除いている。  
もしうまくいったとしても、情報を削ってまで統一するのはちょっともったいない気がする。

この時点で`&str`を`Address`に変換するメソッドを追加したほうが良いという判断をしているのだが、
それはそれとして1つの引数でどちらでも受け取るようにする方法は調べておきたい。

## ではどう書けば良い？

DeepWikiのコードをいじったが、どうにもわからない。

### Claude案

[Claude](https://claude.ai/share/a3933f2e-e721-4f7e-a745-e5b54386a34d)でようやくそのまま動くコードが紹介された。
さすがClaude。さすクロだ。

#### Claude案1

トレイトを作って`self`から`Address` or エラーの戻り値を持つメソッドを用意しておき、
`impl IntoAddress for &str`は`&str`からの変換、
`impl IntoAddress for Address<NetworkChecked>`は所有権あり`Address`からの、
`impl IntoAddress for &Address<NetworkChecked>`は参照`Address`からの変換を実装している。  
呼び出す方は引数を`addr: impl IntoAddress`としておき`addr.into_address()?`を呼ぶと`Address`が戻ってくる。

はー、こうやるんですか。  
説明されると確かにそうなるとはわかるのだが、最初からこれは導き出せないな。  
既定のトレイトではないので引数を追加することも可能だ。

ときどきある、所有権を渡そうと参照だろうと受け付けるタイプの書き方もこれということだ。
単純に参照しか使わないならそう指定するけど、どれで受け付けても結局変換が必要だったらありなのかな。

#### Claude案2

これはちょっと方向性が違うかな。方向性というか私のやりたいこととは異なる。
`fn inspect_address<V: NetworkValidation>(addr: &Address<V>) { ... }`と[NetworkValidation](https://docs.rs/bitcoin/latest/bitcoin/address/trait.NetworkValidation.html)トレイトを使うことで
`NetworkChecked`でも`NetworkUnchecked`でも受けられるようにするというものだ。

Bitcoinアドレスという文字列にするからネットワークという概念が必要になるが、
トランザクションというバイナリデータの中ではそういう情報は持っていない。
"Checked"は`bc1`だの`bcrt1`だの、あるいは`1`で始まったり`m`や`n`で始まったりだののネットワーク情報を含んで間違っていないアドレス。
`Address`は正式には`Address<V = NetworkChecked>`となっていて、デフォルトはそちら側だ。  
秘密鍵や公開鍵から計算したりトランザクションのバイナリデータから直接引っ張ってきたアドレスは`NetworkUnchecked`だろう。

この関数を作るのはウォレット用で、ウォレットは作るときにネットワークを指定することになるので深く考えていなかった。
グローバル変数のような形でどこからでも参照するようにしても良いけど、あまり行儀よくないしトランザクションを作るだけならUncheckedでもいいしなあ、などと考えてしまう。  
そういうこともあってユーザ側に負担をかけさせないためにもどちらでも受け取ることができるようにする、というのはありなのかもしれない。

#### Claude案3

これはなんとなく想像していたタイプで、`enum`で`&str`か`Address`を決めている。

```rust
pub enum AddressInput<'a> {
    Str(&'a str),
    Addr(&'a Address<NetworkChecked>),
}
```

これは引数を`match`で見分ければ済むだけだと思っていたのだが`From`トレイトの実装もされている。
呼び出し方を`AddressInput::Str(アドレス文字列)`のように囲んであげれば不要なのだが、
要望の「文字列でも`Address`でも受け付ける」にするためそうしたのだろう。

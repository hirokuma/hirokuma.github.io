Rustを使っていると `move` がときどき出てくる。
サンプルコードに書いてあったのをまねしたり、それっぽいコードを書いてエラーになったのでAIに解決を求めたら追加されたり。

そんなんじゃダメだよなと思って調べるのだ。

```rust
fn main() {
    let data = vec![1, 2, 3];
    let closure = move || println!("captured {data:?} by value"); // ★move
    closure();
}
```

## moveなし

その前に `move` がないときの動作を見ておこう。

これは `data` をキャプチャしていて特にエラーもない。

```rust
fn main() {
    let data = vec![1, 2, 3];
    let closure = || println!("captured {data:?} by value");
    println!("{:?}", data);
    closure();
}
```

`mut` を付けクロージャの中でキャプチャする値を更新しようとすると、クロージャの定義前はOKだがそのあとにあるとコンパイルエラーになる。

```rust
fn main() {
    let mut data = vec![1, 2, 3];
    data.push(4); // ★ここならエラーにならない
    let closure = || println!("captured {data:?} by value");
    data.push(5); // ★ここはエラーになる
    println!("{:?}", data);
    closure();
}
```

クロージャの中で更新しようとすると、更新する行ではなくクロージャを呼び出すところがコンパイルエラーになった。

```rust
fn main() {
    let mut data = vec![1, 2, 3];
    data.push(4);
    let closure = || {
        data.push(5);
        println!("captured {data:?} by value");
    };
    println!("{:?}", data);
    closure(); // ★ここがエラーになる
}
```

それはクロージャを `mut` にすることで解消できる。

```rust
fn main() {
    let mut data = vec![1, 2, 3];
    data.push(4);// ★これはOK
    let mut closure = || {
        data.push(5);
        println!("captured {data:?} by value");
    };
    // println!("{:?}", data); // ★ここはエラーになる
    closure();
}
```

## move

クロージャのときにしか使われない予約語だ。

* [move - Rust](https://doc.rust-lang.org/std/keyword.move.html)

書いてあるとおりだが、クロージャの中で参照している `data` はそれより後ろだと使えなくなる。
なにを `move` するかというような指示もないので、クロージャの中で参照した変数全般が対象になるのだろうか。

```rust
fn main() {
    let data = vec![1, 2, 3];
    let closure = move || println!("captured {data:?} by value");
    println!("{:?}", data); // ★ここでコンパイルエラー
    closure();
}
```

では `move` したならクロージャ内で変更できるかと思ったが、それもダメなようだ。

```rust
fn main() {
    let mut data = vec![1, 2, 3];
    let closure = move || {
        data.push(5);
        println!("captured {data:?} by value");
    };
    closure(); // ★ここがエラーになる
}
```

これはクロージャが `mut` でないとダメだそうだ。
`move` なしの最後の例とほとんど同じで動作も同じに見えるのだが違うのだ。

```rust
fn main() {
    let mut data = vec![1, 2, 3];
    data.push(4); // ★これはOK
    let mut closure = move || {
        data.push(5);
        println!("captured {data:?} by value");
    };
    // data.push(6); // ★ここはエラーになる
    closure();
}
```

### 違い

`move` のありなしで違うのはクロージャが呼ばれたあと。  
`move` なしならキャプチャした変数の所有権は残っているので、クロージャがそれ以上呼ばれない時点で使えるようになる。

```rust
fn main() {
    let mut data = vec![1, 2, 3];
    data.push(4);
    let mut closure = || {
        data.push(5);
        println!("captured {data:?} by value");
    };
    // println!("{:?}", data); // ★これはエラーになる
    closure();
    println!("{:?}", data); // ★これはOK
}
```

`move` するともう呼び出し元には所有権が残っていないので使うことはできない。

```rust
fn main() {
    let mut data = vec![1, 2, 3];
    data.push(4);
    let mut closure = move || {
        data.push(5);
        println!("captured {data:?} by value");
    };
    // println!("{:?}", data); // ★これはエラーになる
    closure();
    println!("{:?}", data); // ★ここもエラーになる
}
```

### moveいるの？

実のところクロージャは `move` がなくてもクロージャの中でのキャプチャした変数の使い方によってうまいこと扱いを変えようとする。
その「うまいこと」の中には所有権を奪うことまで選択肢に含まれているそうだ。
`Fn`, `FnOnce`, `FnMut` が実装によって判断されるのと同じと思えば良さそう。

そうなると `move` はなくてもなんとかなるんじゃないの？と思う。
`move` を付けると明示的に所有権が移動することになるというだけでよいことがなさそうだからだ。

それがいつ必要かというと、非同期というか別タスクというか、動作する先がいつ終わるかはっきりしないので「もううちの子ではありません」と言い切りたい場合だ。

* [参照をキャプチャするか、所有権を移動するか](https://doc.rust-jp.rs/book-ja/ch13-01-closures.html#%E5%8F%82%E7%85%A7%E3%82%92%E3%82%AD%E3%83%A3%E3%83%97%E3%83%81%E3%83%A3%E3%81%99%E3%82%8B%E3%81%8B%E6%89%80%E6%9C%89%E6%A8%A9%E3%82%92%E7%A7%BB%E5%8B%95%E3%81%99%E3%82%8B%E3%81%8B)

```rust
fn main() {
    let mut data = vec![1, 2, 3];
    data.push(4);
    std::thread::spawn(move || {
        data.push(5);
        println!("captured {data:?} by value");
    }).join().unwrap();
}
```

tokio::spawn() でも `move` がないとダメだ。
というよりは `async{}` ブロックの中でキャプチャする時点で `move` しなさいということかしらね。

> async block may outlive the current function, but it borrows `data`, which is owned by the current function
> async blocks are not executed immediately and must either take a reference or ownership of outside variables they use

```rust
#[tokio::main]
async fn main() {
    let mut data = vec![1, 2, 3];
    data.push(4);
    tokio::spawn(async move {
        data.push(5);
        println!("captured {data:?} by value");
    }).await.unwrap();
}
```

大雑把な感想だが、`move` の詳細を把握しておかないとRustで潜在的なバグがあるコードを書いてしまう、ということにはならなさそうだ(コンパイルエラーになるので)。

邪道な？TOMLファイル読み込みをいろいろやっていた。

やっていたというか、自然に訳が分からないことをやっていたというやつだ。

## 2回に分ける

これは、先にライブラリAを作り、ライブラリAの動作確認アプリのためにTOMLファイルを読み込んでいて、
その後にライブラリBを作って別のTOMLファイルを読み込んだ動作確認アプリを作り、
最後にライブラリA,Bを使ったアプリを作り、それぞれで使っていたTOML読み込み機能をそのまま流用したらこうなった、という感じだ。

```rust
use serde::Deserialize;

#[derive(Deserialize, Debug)]
struct AbcBoolConfig {
    type_abc: bool,
}

#[derive(Deserialize, Debug)]
struct AbcConfig {
    url: String,
}

#[derive(Deserialize, Debug)]
struct DefBoolConfig {
    type_def: bool,
}

#[derive(Deserialize, Debug)]
struct DefConfig {
    name: String,
}

#[derive(Deserialize, Debug)]
struct Config1 {
    abc: AbcConfig,
    ghi: AbcBoolConfig,
}

#[derive(Deserialize, Debug)]
struct Config2 {
    def: DefConfig,
    ghi: DefBoolConfig,
}

fn main() {
    let toml_str = r###"
[abc]
url = "http://fuga"

[def]
name = "taro"

[ghi]
type_abc = true
type_def = false
    "###;

    let config: Config1 = toml::from_str(toml_str).unwrap();
    println!("{:?}", config);
    let config: Config2 = toml::from_str(toml_str).unwrap();
    println!("{:?}", config);
}
```

実行

```log
Config1 { abc: AbcConfig { url: "http://fuga" }, ghi: AbcBoolConfig { type_abc: true } }
Config2 { def: DefConfig { name: "taro" }, ghi: DefBoolConfig { type_def: false } }
```

動いていたので特に気にせずにいたのだが、アプリの実装が増えてTOMLの項目が増え、
TOMLになくてもデフォルト値を使いたかったり、細かいことは忘れたが何かの変更を加えたところでどうしようもなくなってしまった。  
共通で使用しているテーブル名と同じキー名を使ってしまったのがダメだったのだとは思う。
そうなったのも、TOMLファイルを読み込む `struct` がどんどん階層深くなっていって把握できていなかったからだ。

最終的には、ライブラリごとにテーブルに階層をもたせることにした。
そうすればいちいち `toml::from_str()` を呼び出さずにトップレベルだけ呼び出せば済んだ。  
イメージとしてはこんな感じか。

```toml
[abc]
url = "http://fuga"

[abc.ghi]
type_abc = true

[def]
name = "taro"

[def.ghi]
type_def = false
```

## AIにつくってもらおう

このTOMLファイルをGeminiにあたえて `struct` を作るよう依頼した。
ちゃんとコードを作ってくれたので、この程度であれば頭を悩ませるよりは任せてしまってもよいかもしれん。

```rust
use serde::Deserialize;

#[derive(Deserialize, Debug)]
struct Config {
    abc: AbcConfig,
    def: DefConfig,
}

#[derive(Deserialize, Debug)]
struct AbcConfig {
    url: String,
    ghi: AbcGhi,
}

#[derive(Deserialize, Debug)]
struct AbcGhi {
    type_abc: bool,
}

#[derive(Deserialize, Debug)]
struct DefConfig {
    name: String,
    ghi: DefGhi,
}

#[derive(Deserialize, Debug)]
struct DefGhi {
    type_def: bool,
}

fn main() {
    let toml_str = r#"
        [abc]
        url = "http://fuga"

        [abc.ghi]
        type_abc = true

        [def]
        name = "taro"

        [def.ghi]
        type_def = false
    "#;

    let config: Config = toml::from_str(toml_str).expect("Failed to parse TOML");
    println!("{:#?}", config);
}
```

実行

```log
Config {
    abc: AbcConfig {
        url: "http://fuga",
        ghi: AbcGhi {
            type_abc: true,
        },
    },
    def: DefConfig {
        name: "taro",
        ghi: DefGhi {
            type_def: false,
        },
    },
}
```

こういうファイルのパースは、変数名を合わせたり `#[serde()]` で修飾しないといけなかったりとルールが多いので、
そこに時間を割かなくて良いと判断したなら作ってもらって良いと思う。  
AIに頼んでしまいたいところと、いちいちAIに頼んで作ってたら仕事にならないってところがあるので、うまいことやっていきたいものだ。

* [main.rs](https://github.com/hirokuma/bdk-script-example2/blob/65273c3704fa2184f7caa371a32219a9a54b2c54/src/main.rs)

preimageルート用のコードをコピーして書き換えていっただけなので似たようなコードになってしまった。  
違いは`OP_CHECKSEQUENCEVERIFY`の判定部分。  
`check_older()`の引数でスクリプトの`OP_CSV`の値のほうが渡されるので、実際のトランザクションのsequence値が条件を満たしているかチェックする。

```rust
impl Satisfier<PublicKey> for RefundSatisfier {
    ...略...

    fn check_older(&self, csv: LockTime) -> bool {
        csv.to_consensus_u32() >= self.sequence
    }
}
```

トランザクションのsequence値は`struct RefundSatisfier`のフィールドとしてもらうようにしたので自分で与えているだけだ。
今回はvinが1つしかない前提なので`input[0]`固定としたが、まあよかろう。

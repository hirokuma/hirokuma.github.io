未だに苦手なトレイト。

インターフェース的なものだし、特にいらなくてもとりあえず大きめの`struct`には作っておいて、実装するときは`struct`の方だけ使っておけばよかろうと考えた。  
が、そうすると`struct`のインスタンスは作ることができたがメソッドが呼び出せない。
なんでよー、と調べると`trait`も`use`しないといけないということだった。

それはそれでなんでインスタンスを直接使っているのに`trait`がいるのよー、と調べると`trait`が違えば同じ関数名でも定義できるからだということだった。
どちらかというと、使いたい`trait`が同じ関数名を使っていたとしても大丈夫なようにそうなっているという方が正しいか。
C++の`namespace`的なものかね。

呼び出すときはドットでつなげるのではなく第1引数の`self`などをインスタンスにする。
`self`系の引数がなければ`as`で関連付けるようだった。

```rust
trait A {
    fn hello();
}

trait B {
    fn hello(&self);
}

struct Abc {
    name: String,
}

impl A for Abc {
    fn hello() {
        println!("A::hello");
    }
}

impl B for Abc {
    fn hello(&self) {
        println!("B::hello({})", self.name);
    }
}

fn main() {
    let abc = Abc { name: "Tom".to_string() };
    <Abc as A>::hello();
    B::hello(&abc);
}
```

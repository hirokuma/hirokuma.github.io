Rustの本を読んでいたのだが、エラー関連のところで「Fromトレイトの実装があると自動で呼び出すので便利」みたいな記述があった。
よく読まずに(というよりもFromトレイトの話はそんなに書いてなかった)、面白そうなので使ってみようとしたのだが、何をどうしたものかわからん。

## 自動で呼び出す機能はない

"from" みたいなのは検索しづらいので、さっさとGeminiに確認。

そういう機能は、ない。  
以上。

## しかし、そう見えるのはある

あんたがいいたかったのはこれじゃないの？と言ってきたのが `?` が使われた場合。  
戻り値の型が `Result<T, E>` で戻ってきたのが `Err(E)` だったときに `?` があると処理はそこで終わってエラーをそのまま上に投げ返すのだが、
上に投げ返すエラーの型が `E` ではなく `F` で、`F` が Fromトレイトを実装して `from<E>` が呼び出せるなら `E` から `F` の変換をして返すそうだ。

確かに、本にもそう書いてある。
Fromトレイトがあるとこんなこともできるんですよ、というのをエラーの処理ついでに説明しているのかと思っていたが、
このしくみがエラーのときにしか動作しないから説明しているということだな。

```rust
use std::fmt::{Display, Formatter};

#[derive(Debug)]
pub struct MyError(String);

impl Display for MyError {
    fn fmt(&self, f: &mut Formatter<'_>) -> std::fmt::Result {
        write!(f, "MyError happened: {}", self.0)
    }
}

impl std::error::Error for MyError {}

impl From<String> for MyError {
    fn from(value: String) -> Self {
        Self(format!("convert from: {}", value))
    }
}

fn func() -> Result<u32, String> {
    Err("hello my error".to_owned())
}

fn main() -> Result<(), MyError> {
    func().inspect_err(|e| eprintln!("main: {}", e)).map(|_v| Ok(()))?
}
```

実行すると、`func()` はエラーとして `String` を返しているだけで `main()` の `eprintln!()` ではその文字列を出力している。
しかしおしりに `?` があるため `From<String> for MyError` の `from()` が呼ばれて "convert from:" がひっつき、
コンソールに出力するときは `Display for MyError` が呼ばれて "My Error happened:" がひっついた、ということか。

```bash
main: hello my error
Error: MyError("convert from: hello my error")
```

うーん、便利そうな気はするが私だと `.map_err()` でやってしまいそうな気がする。
ああでも自作ライブラリで `?` を使っているところにいちいち `.map_err()` を埋め込むよりも良い気がしてきた。
ああいや、`.map_err()` を書くとついでに `error!()` などとログ出力できるというメリットがあるか。
`from()` が自動で呼ばれてもどのファイルの何行目でエラーになったかわからないのでログを出したいのだ。

エラーを返すところでログを出したいけれどもいちいち書くのが面倒なのでデバッグの時だけでも自動でやってくれないかなあ、
と本題とぜんぜん違うことを考えてしまった。

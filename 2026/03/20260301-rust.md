Rustで「この `Option<T>` が `None` だったらアーリーリターンさせよう」と考えることがしばしばある。
そういうときに let-else文を使うことができる。

* [rust: Result<T, E> と Option<T> - hiro99ma blog](https://blog.hirokuma.work/langs/rust/result_option.html#let-else--none-%E3%81%AA%E3%82%89-early-return-%E3%82%92%E3%81%9D%E3%81%86%E3%81%A7%E3%81%AA%E3%81%84%E3%81%AA%E3%82%89%E7%B6%99%E7%B6%9A)

```rust
fn main() {
    let f = || -> Option<String> { None };
    let Some(v) = f() else {
      eprintln!("I am None!");
      return;
    };
    println!("v = {}", v);
}
```

実行

```
I am None!
```

## Result で使い勝手が良いやつはないか

`Result<T,E>` でも let-else文は使えるのだが、`Result` だと `Err` の方を確認したいよなあ。
そう思うと `Option` で使うことが多いように思う。
まあ、それは `unwrap_or()` みたいなのもそうだろう。

エラーのときはアーリーリターンしたいけどエラー値も使いたい、という場合は `match` を使うか、

```rust
match func() {
  Ok(v) => ...,
  Err(e) => ...
}
```

`?` と `.inspect_err()` や `.map_err()` などを組み合わせるか。
`.inspect_err()` は `.map_err()` のエラー値変更無し版だ。
なので `.map_err()` でエラーを変換せず返せば `.inspect_err()` と同じ事ができる。  
エラーの内容を把握しつつ、終わりが `?` なのでアーリーリターンできる。

```rust
fn main() -> Result<(), String> {
    let f = || -> Result<String, &str> { Err("NG") };
    let v = f().inspect_err(|e| eprintln!("This function return error: {e}"))?;
    println!("v = {}", v);
    Ok(())
}
```

```rust
fn main() -> Result<(), i32> {
    let f = || -> Result<String, &str> { Err("NG") };
    // エラーを変換せずそのままeを返しても良い
    let v = f().map_err(|e| {eprintln!("This function return error: {e}"); -123})?;
    println!("v = {}", v);
    Ok(())
}
```

* [std::result - Rust](https://doc.rust-lang.org/std/result/#transforming-contained-values)
